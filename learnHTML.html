<HTML Basics:>

An HTML document is a text file that contains text and markups called tags. So, simple text editor such as Notepad can be used to create and edit HTML files. The only requirement is that the documents must have the extension .htm or .html
An HTML page is all about tags. Tags are similar to instructions or codes. Web browsers know their meaning and interpret them to render text, images, and other elements on the screen properly. The set of HTML tags is fixed and is standardized by w3c. Invalid tags are displayed on the screen as they are.

<8 . Table:>

Tabular structure was first introduced in HTML 3.2. It is sometimes useful to display data in a two-dimensional tabular format. Tables are also powerful tools for formatting your webpages. You may have seen many web pages with a navigation panel at the left.Perhaps these webpages use tables.

A table is created using three basic tags:table,tr, and td. The tag table is the container of the whole table. The opening tag <table> tells the browser that it is to be created. Table-specific properties are also defined here. The closing tag </table%gt; tells the browser that it is the end of the table.Everything else is contained within the <table> and </table> tag pair. So, a table always look like this.

	<table>
	.............
	.............
	</table>

The following table describes the set of tags used to create tables along with their meaning:
HTML table tags
TagMeaning
tableRepresents the whole table
trRepresents a row
tdRepresents a cell in a row
thColumn header
captionTitle of the table

A table consists of a number of rows each of which is made up of a number of cells. The tag <tr> cretes a row. The total number of rows in a table is equal to the number of <tr> elements. The opening tag <tr> and closing tag </tr> indicate the beginning and ending of a table row, respectively. 

	<table>

	<tr>.......</tr>
	<tr>.......</tr>
	........
	<tr>.......</tr>
	</table>

A table or a row cannot contain data. To add data, cells are created within a row. Each cell in a row is defined by the tag <td>. The number columns in a row is equal to the number of <td> elements. The content of a cell is written in the <td> and </td> tag pair. Textual as well as graphical content can be used in cell. Usual tags such as <p>,<br>,<i>,<b>, & <u> may be used within the cell.

<table>
<tr><td>...</td><td>...</td><td>...</td></tr>
<tr><td>...</td><td>...</td><td>...</td></tr>
........
<tr><td>...</td><td>...</td><td>...</td></tr>
</table>

Two other tags are defined: <th> and <caption>. To add a column header ,the <th> tag is used. The title of table is added using the <caption> tag.

Let us first create a simple 2x2 table to display the marks of two students in two different subjects.

sample program

Result

<body>
<table>
<tr><td>82</td><td>80</td></tr>
<tr><td>80</td><td>75</td></tr>
</table>
</body>
</html>

<9 . Forms:>

The HTML <form> tag is a very important tag, which creates a section in your HTML document. It is used to collect information from visitors. It contains special elements called controls. HTML provides several control elements such as buttons, textboxes, passwordfields, checkboxes, radiobuttons, selectionboxes, hiddenfields, etc. Visitors enter data using these control elements.These data are then usually sent (submitted) to a program at the server side.
The tag <form> is a container tag and is created as follows:
<form>
<!---Form elements, markups and other contents go here --->
</form>
Optional attributes of <form> tag:
action-specifies the target URL, which handles the data.
method-specifies the HTTP method to be used. It can have values GET or POST.
name-used to refer to the form from scripts or style sheets.

9.0:Form Elements

Form elements are also called Control elements.Control elements are created using the <input> tag. It has an attribute type that specifies the type of the input. There are 10 input types: text, password, checkbox, radio, button, submit, reset, hidden, file, & image.
Attributes of the input tag
AttributeMeaning
typeSpecifies the type of a control element. Can have values text,password,checkbox,radio,submit,reset,hidden,file,image,button. Its default value is text
nameSpecifies the name of a control element. Submitted as a part of form.
valueSpecifies the initial value of the control element, e.g.,text for the text control, caption for submit, reset and buttton elements, etc. Optional except for check box and radio button.
sizeSpecifies the maximum number of characters that can be viewed in case of a text or password field. Otherwise, it specifies the width of the control in pixels.
maxlengthSpecifies the maximum number of characters a user can enter in a text or password field.
checkedUsed for check boxes and radio buttons. Indicates that the control is on. The browser ignores for all other controls.
readonlyUsed for text, password, and file controls. If set, prevents changes to the controls.
disabledIf set, disables controls from the user input.

9.10.Action buttons

There are 2 types of buttons that we use for almost all forms: submit button and reset button.
Submit button: Submit button is used to send information to the server script whose URL is specified by the action attribute of the <form> tag. A submit button is created as follows:
<input type="submit" value="send">
This code creates a button with caption "send" which when clicked, sends form's data to the server.
Reset button :When a reset button is clicked, form fields are assigned their default initial values. The attribute type species that it is a reset button and value specifies the text to be displayed on the reset button.
Clicking on a reset button accidentally erases the values supplied by the users to the form fields. To avoid this situation, a handler may be added as follows:

sample program

Result

<html>
<body>
<form action="http://www.gprecpaper.weebly.com">
<input type="submit" value="send">
<input type="reset" value="Restore Defaults"
onclick="return confirm('Do you really want to reset the form?')">
</form>
</body>
</html>

10.1.Frameset element

The <frameset> element is put in the parent HTML document without any <body> tag. It contains <frame> elements each of which creates a frame. The element <frameset> defines the characteristic of the frames such as the number of frames to be created, their orientation, width, height, etc.

10.2.Frame Element

The element <frame> specifies the HTML document to br loaded in each frame. It has several attributes discussed as follows:

  • name-refers to the name of the frame.

  • src-specifies the location of the initial webpage to be loaded.

  • noresize-if present, indicates that the frame is not resizable.

  • scrolling-specifies the scroll information about the frame. It can have the following values.

    • auto-specifies that scrollbars will be added as when necessary.

    • yes-specifies that scrollbars are to be added always.

    • no-specifies that no scrollbars will be provided.

  • frameborder-specifies the border information about the frame. It can have the following values.

    • 1-indicates that the border is drawn.

    • 0-indicates that the frame will not have any border.

  • marginwidth-specifies the amount of space in pixels to be added between the frame's contents in its left and right margins.

  • marginheight-specifies the amount of space in pixels to be added between the frame's contents in its top and bottom margins.

10.3.Specifying Target

The target attribute may be set for elements that create links such as <a>, <link>, <area>, <form>, etc. The value of target attribute must be an existing frame name.

First we define a frameset document main.html as follows.
<html>
<frameset rows="20%,80%">
<frame name="top" src="top.html">
<frame name="bottom" src="body.html">
</frameset>
</html>
This divides the window into two frames horizantally. In the top frame, HTML document top.html is loaded. It looks like this:
<html>
<body>

<a href ="learnHTML.html" target="bottom">HTML for Beginners</a>	
<a href ="body.html#4" target="bottom">ABOUT US</a>
</html>
